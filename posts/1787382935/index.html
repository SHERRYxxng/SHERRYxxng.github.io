工作日志 2026-05-22

前端改动

1. 后端代理地址切换

  • 文件: dandelion-ui/vue.config.js
  • 改动: 将开发环境的 baseUrl 从 172.16.1.7:9998（测试环境）改为 localhost:8080（本地环境）
  • 原因: 本地开发需要连接本地后端

2. resource-table 列表数据兼容 records 格式

  • 文件: dandelion-ui/src/views/cmdb-config/components/resource-table.vue (第300行)
  • 改动: getList() 方法中补充 response.data?.records 的数据读取
  • 原代码: response.data?.instanceList || response.data?.list || response.data || response.rows || []
  • 新代码: response.data?.instanceList || response.data?.list || response.data?.records || response.data || response.rows || []
  • 原因: 二类节点 API 返回 data.records 格式的数据

3. CMDB 密钥管理 - 启用二类节点类型

  • 文件: dandelion-ui/src/views/cmdb-config/key.vue (第72行)
  • 改动: 移除节点类型下拉中 CTYUN_LEVEL2 的禁用状态
  • 原代码: :disabled="dict.value === 'CTYUN_LEVEL2'"
  • 新代码: 去掉该属性
  • 原因: 需要支持创建二类节点账号，选择后自动显示 projectID 和 endpoint 字段

4. REDIS/MySQL/ELB 字典数据缺失导致表格不显示

  • 问题: REDIS/MySQL/ELB 页面点进去看不到数据，API 有返回但表头字典为空
  • 修复: 在本地数据库插入表头字典数据
  • 涉及字典类型:
    • cmdb_ctyun_redis_details_header（20个字段）
    • cmdb_ctyun_mysql_list_detail_header（18个字段）
    • cmdb_ctyun_elb_list_header（8个字段）
  • 原因: 数据库缺少字段定义，前端无法渲染列头
  • 需要: 后端切换到 local 配置，重启后生效

5. VM 二类节点 tab 禁用修复

  • 文件: dandelion-ui/src/views/cmdb-config/components/resource-table.vue (第5行)
  • 改动: 移除 <el-tab-pane label="二类节点"> 上的 disabled 属性
  • 原因: disabled 导致无法点击切换二类节点 tab

后端改动

1. 二类节点表格无数据修复 — CTYunLevel2EcsDetails.java

  • 添加 getInstanceID() 返回 ecsId，兼容前端 :row-key
  • 新增 accountAlias 字段标记来源账号

2. 二类节点同步改为多账号 — CTYunLevel2EcsServiceImpl.java

  • 从数据库读取所有 CTYUN_LEVEL2 类型账号，遍历同步，不再硬编码
  • 注入 CTYunAssetAccountMapper 代替 CTYAssetAccountService，打破循环依赖
  • InterceptorIgnoreHelper 用 try-finally 包裹，确保租户隔离不泄露
  • 排查 getEcsList 中 selectBatchIds 异常时 clearIgnoreStrategy 未执行的 bug
  • 定时任务 doSync 和 getEcsList 查询账号时忽略多租户拦截

3. 一类节点去重 & 来源账号

  • CTYunEcsInstanceDetailMapper.xml: 去重子查询 + LEFT JOIN asset_account
  • EcsInstanceDetailsReturnObjVo.java: 新增 accountAlias 字段
  • CTYunEcsInstanceDetailServiceImpl.java: 去掉 batchId 过滤
  • 发现一类节点三个账号（长新云/多云/我的）instance_id 完全重叠，去重后仅20台

4. 资源数量合并 — CTYunCommonServiceImpl.java

  • VM 计数 = 一类节点(asset_main) + 二类节点(level2)
  • 改为 MyBatis Mapper 查询（回退 JdbcTemplate 方案），遵循多租户插件规范
  • 二类节点计数移至 ecs_instance_details 查询之后，避免被覆盖
  • 一类节点计数去掉 batchId 过滤，启用 Mapper XML 中的去重子查询（按 instance_id 取最新记录）
  • 最终 VM = 一类(去重后) + 二类

5. discover 多账号测试 — CTYunLeve2EcsController.java

  • 添加 @Anonymous，支持多 AK/SK × 多端点测试

6. 数据库变更

变更 说明
清空网大数据 TRUNCATE cmdb_ctyun_level2_ecs_details
新增账号 多云(一类)、我的(一类+二类)、网大(二类)
字典项 一类 accountAlias、二类 accountAlias
清理残留 删除已删除账号的39条数据
清理 NULL account_id 旧同步代码残留的1条重复数据

问题排查记录

循环依赖

  • CTYunAssertAccountServiceImpl ↔ CTYunLevel2EcsServiceImpl 相互引用
  • 解决：CTYunLevel2EcsServiceImpl 改用 Mapper 直查，绕过 Service 层

定时任务租户异常

  • @Scheduled 定时任务无登录上下文，多租户插件报 获取租户ID异常
  • 解决：查询时用 InterceptorIgnoreHelper.handle() 忽略租户拦截

InterceptorIgnoreHelper 未清理

  • getEcsList 中 selectBatchIds 抛异常时 clearIgnoreStrategy() 不执行
  • 后果：后续所有查询跳过租户过滤，可能导致数据泄露
  • 解决：全局用 try-finally 包裹

6. 后端切换 local 配置

  • 文件: dandelion-backend/dandelion-admin/src/main/resources/application.yml (第100行)
  • 改动: active: test → active: local
  • 原因: 切换本地数据库和 Redis，使用完整数据（199台VM而非45台）

7. 字典乱码修复

  • 问题: cmdb_ctyun_redis_details_header / cmdb_ctyun_mysql_list_detail_header / cmdb_ctyun_elb_list_header 的 dict_label 中文乱码
  • 原因: 插入数据时 MySQL 客户端字符集不匹配，UTF-8 字节被当作 latin1 存储
  • 修复: 直接 SQL UPDATE 修正 47 条记录的 dict_label

8. 二类节点资源发现端点

  • 文件: CTYunLeve2EcsController.java
  • 端点: GET /cmdb/ctyun/level2/ecs/discover-resources
  • 功能: 遍历所有 CTYUN_LEVEL2 账号，调用 CTECS OpenAPI + 各服务 SDK API 检查 Redis / MySQL / ELB 资源情况
  • 兜底: CTECS OpenAPI 鉴权失败时使用已知区域列表（cn-sh1, cn-north1 等）继续尝试
