feature/CMDB 与 test 分支合并检查报告

API 接口变更

新增接口

1. 合并一类+二类云主机列表

  • 入参: { pageNo: 1, pageSize: 10, commonQuery: "" }
  • 出参: { instanceList: [...], total: 199, pageNo, pageSize }
  • 功能: 合并一类（去重20台）+ 二类（179台）统一分页返回

2. 二类节点资源发现

  • 入参: 无
  • 出参: { "账号别名": { "CTECS_OpenAPI": {...}, "REDIS(DCS)": {...}, "MYSQL(RDS)": {...}, "ELB": {...} } }
  • 功能: 遍历所有 CTYUN_LEVEL2 账号，检查在各服务下是否有资源

3. 云硬盘列表（框架已建，API待验证）

  • 入参: { pageNo, pageSize, commonQuery }
  • 状态: 框架搭建完成，EVS API 端点待确认（猜测 https://evs-global.ctapi.ctyun.cn）

修改接口

4. 资源概览 GET /cmdb/common/CTYun/resources/list

  • 行为变化：
    • VM 计数 = 一类(去重后 20) + 二类(table COUNT)
    • 计数仅来自 DB 已同步数据（VM/MYSQL/REDIS/ELB）
    • Volume/VPC/NAT 等字段返回 0（未同步）
    • 不再调用 CTECS OpenAPI 实时查询（因超时已移除）

数据库变更

新建表

修改表

  • cmdb_ctyun_level2_ecs_details 已通过 accountId 关联资产账号

需手动执行

  • V20260522001__add_cmdb_volume_table.sql Flyway 迁移已在本地跑过
  • 合并后需要在 test 环境执行此迁移

application.yml 变更

active: test → active: local（仅本地开发，合并时不应带上）

合并注意事项

  1. application.yml — 本地使用了 active: local 配置，合并时必须保留 test，不能覆盖
  2. Flyway 迁移 — V20260522001__add_cmdb_volume_table.sql 需要在 test 环境执行
  3. 云硬盘功能 — API 端点待验证，EVS 调用可能失败（端点猜测）：不影响现有功能
  4. 新增 Mapper — CTYunVolumeMapper 及 XML 需要被 MyBatis 扫描到
  5. .gitignore — 新增了 application-local.yml 排除规则
